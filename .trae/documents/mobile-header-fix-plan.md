# 移动端显示优化计划

## 问题描述
当前移动端显示存在混乱问题，需要确保点击进模块后隐藏header组件，与网页端保持一致。

## 问题分析
1. 移动端header组件在某些情况下没有正确隐藏
2. 页面状态管理在移动端可能存在逻辑问题
3. 需要确保移动端和网页端的header显示逻辑一致

## 实施步骤

### 步骤1: 检查当前移动端Header显示逻辑
- 分析Header组件中的移动端判断逻辑
- 检查usePageStore状态管理在移动端的应用
- 验证页面类型检测逻辑在移动端的表现

### 步骤2: 修复移动端Header隐藏逻辑
- 确保移动端也能正确识别博客文章、关于、联系页面
- 统一移动端和网页端的隐藏逻辑
- 修复可能的状态管理问题

### 步骤3: 优化移动端菜单交互
- 确保移动端菜单展开时不会意外显示header
- 优化移动端菜单的显示和隐藏逻辑
- 确保页面切换时移动端菜单正确关闭

### 步骤4: 测试移动端显示效果
- 在不同页面测试移动端header显示
- 验证页面切换时的header隐藏效果
- 确保移动端触摸交互正常

### 步骤5: 验证响应式断点
- 检查移动端断点设置（sm:hidden等）
- 确保在不同屏幕尺寸下显示正常
- 优化移动端布局避免元素重叠

## 预期结果
- 移动端header显示逻辑与网页端完全一致
- 进入博客文章、关于、联系页面时header正确隐藏
- 移动端菜单交互流畅，无显示混乱问题
- 所有页面在移动端都有良好的用户体验
